I'll explain this R code in detail. This function implements a modified version of the Connectivity Map (CMap) scoring algorithm, which is commonly used in drug repurposing and gene expression analysis.

Let me break down the key components and functionality:

  1. Function Purpose:
cmap_score_new <- function(sig_up, sig_down, drug_signature)

This function calculates a connectivity score between a drug signature and disease signature (split into up-regulated and down-regulated genes). The function takes three parameters:

  • sig_up: Up-regulated genes in the disease signature
  • sig_down: Down-regulated genes in the disease signature
  • drug_signature: The drug's gene expression signature
  1. Initial Setup:
num_genes <- nrow(drug_signature)
ks_up <- 0
ks_down <- 0
connectivity_score <- 0

The function initializes variables for:

  • Total number of genes in the drug signature
  • Kolmogorov-Smirnov (KS) statistics for up and down regulated genes
  • The final connectivity score
  1. Data Preparation:
drug_signature[,"rank"] <- rank(drug_signature[,"rank"])
up_tags_rank <- merge(drug_signature, sig_up, by.x = "ids", by.y = 1)
down_tags_rank <- merge(drug_signature, sig_down, by.x = "ids", by.y = 1)

This section:

  • Re-ranks the drug signature to handle any changes in gene ID mapping
  • Merges the drug signature with up and down regulated disease genes
  • Creates sorted positions of the matched genes
  1. KS Statistic Calculation: The function calculates modified Kolmogorov-Smirnov statistics separately for up and down regulated genes. For each direction (up/down), it:
if(num_tags_up > 1) {
    a_up <- max(sapply(1:num_tags_up,function(j) {
      j/num_tags_up - up_tags_position[j]/num_genes
    }))
    b_up <- max(sapply(1:num_tags_down,function(j) {
      up_tags_position[j]/num_genes - (j-1)/num_tags_up
    }))
  • Checks if there are enough genes (>1) to calculate the statistic
  • Calculates two components (a and b) using the empirical cumulative distribution
  • Takes the maximum deviation in both positive and negative directions
  • Assigns the larger absolute value with appropriate sign
  1. Final Score Calculation:
if (ks_up == 0 & ks_down != 0){ #only down gene inputed
    connectivity_score <- -ks_down 
} else if (ks_up !=0 & ks_down == 0){ #only up gene inputed
    connectivity_score <- ks_up
} else if (sum(sign(c(ks_down,ks_up))) == 0) {
    connectivity_score <- ks_up - ks_down # different signs
} else {
    connectivity_score <- ks_up - ks_down
}

This section handles different cases:

  • Only down-regulated genes present
  • Only up-regulated genes present
  • Both present with different signs
  • Both present with same signs

The connectivity score indicates:

  • Positive score: Drug signature matches the disease signature
  • Negative score: Drug signature opposes the disease signature
  • Magnitude indicates strength of the connection

This implementation differs from the original CMap approach by:

  1. Supporting input lists with either all up or all down genes
  2. Modifying how the final score is calculated when KS statistics have different signs
  3. Avoiding score enrichment at 0

The results can be used to identify potential drug candidates that might reverse disease states based on gene expression patterns.

Detailed Analysis of CMap Score Function

Function Signature and Input Types

cmap_score_new <- function(sig_up, sig_down, drug_signature)

Input Parameters:

  1. sig_up:

    • Type: data.frame or vector
    • Content: Gene identifiers for up-regulated genes
    • Expected format: Single column with gene IDs that match the ids column in drug_signature
    • Can be empty but must exist

  2. sig_down:

    • Type: data.frame or vector
    • Content: Gene identifiers for down-regulated genes
    • Expected format: Same as sig_up
    • Can be empty but must exist

  3. drug_signature:

    • Type: data.frame
    • Required columns:
      • ids: Character or numeric - Gene identifiers
      • rank: Numeric - Original ranking of genes
    • Structure: Each row represents a gene with its expression rank

Function Components Analysis

1. Initialization Block

5. KS Statistic Calculation

For up-regulated genes (similar structure for down-regulated):

    a_up <- max(sapply(1:num_tags_up,function(j) {
      j/num_tags_up - up_tags_position[j]/num_genes
    }))
    b_up <- max(sapply(1:num_tags_up,function(j) {
      up_tags_position[j]/num_genes - (j-1)/num_tags_up
    }))
    
    if(a_up > b_up) {
      ks_up <- a_up
    } else {
      ks_up <- -b_up
    }
}
  • Inputs:
    • num_tags_up: Integer - Number of matching up-regulated genes
    • up_tags_position: Numeric vector - Sorted ranks
    • num_genes: Integer - Total genes
  • Calculations:
    • a_up: Maximum positive deviation
    • b_up: Maximum negative deviation
  • Output:
    • ks_up: Signed KS statistic

6. Final Score Computation

if (ks_up == 0 & ks_down != 0){ 
    connectivity_score <- -ks_down 
} else if (ks_up !=0 & ks_down == 0){ 
    connectivity_score <- ks_up

    connectivity_score <- ks_up - ks_down

    connectivity_score <- ks_up - ks_down

  • Logic branches:
    1. Only down-regulated genes: Returns negative of down KS
    2. Only up-regulated genes: Returns up KS
    3. Different signs: Subtracts down from up
    4. Same signs: Subtracts down from up

Return Value

  • Type: Numeric
  • Range: Typically between -2 and 2
  • Interpretation:
    • Positive: Drug signature aligns with disease signature
    • Negative: Drug signature opposes disease signature
    • Magnitude: Strength of connection
